I am starting fresh. I need a copy of the repo.

git clone <the path to the repository>

I've made changes to the code on my end and I want to save the state on my machine for later.

git commit -m "your commit message here"

Note: this will only commit files that you have added via git add <filename>. This allows you to commit only files that you want. If you want to commit all the tracked files in your working directory each time, use:

git commit -am "your commit message"

I want to push the changes to the remote server, allowing the other members of my team to see and use them.

git pull origin master (see below for discussion about pull errors)

then

  • git add file1.lisp file2.py (replace with the files you want to add)

  • git commit -m "your message"

  • OR

  • git commit -am "your message" (see above for the difference)

then...

git push origin master

Help! it says I need to pull first!

Yes, you are correct. It is good practice to do a git pull origin master before you start coding for the day, but sometimes we forget, and sometimes someone has pushed something before you. If this happens, run:

git pull origin master
